React API верхнего уровня

Если вы загружаете React из тега <script>, API верхнего уровня доступны на глобальном уровне. Если вы используете ES6 с npm, вы можете написать import React from 'react'. Если вы используете ES5 с npm, вы можете написать var React = require('react').

Общее

Компоненты

Компоненты React позволяют разделить пользовательский интерфейс на независимые, многоразовые фрагменты и работать с каждым отдельно. Компоненты могут быть определены путем подклассификации React.Component или React.PureComponent.

  • React.Component
  • React.PureComponent

Если вы не используете классы ES6, вы можете использовать create-react-class модуль.

Создание элементов React

Мы рекомендуем использовать JSX для описания того, как должен выглядеть ваш пользовательский интерфейс. Каждый элемент JSX вызывает React.createElement(). Обычно вы не используете следующие методы, если используете JSX.

  • createElement()
  • createFactory()

Преобразование элементов

React предоставляет несколько API для управления элементами:

  • cloneElement()
  • isValidElement()
  • React.Children

Фрагменты

React также предоставляет компонент для рендеринга элементов без оболочки.

  • React.Fragment

Refs

  • React.createRef
  • React.forwardRef

Справка

React.Component

React.Component является базовым классом для компонентов React, когда они определены с использованием классов ES6 :

class Greeting extends React.Component {
    render() {
      return 
Hello, {this.props.name}
;
    }
  }

См. Справочник API React.Component для списка методов и свойств, связанных с базовым React.Component классом.

createElement() 

React.createElement(
  type,
  [props],
  [...children]
)
  }

Создайте и верните новый элемент React данного типа. Аргумент типа может быть либо строкой имени тега (например, 'div'и 'span'), типа компонента React (класс или функция) или типа фрагмента React . Код, написанный с помощью JSX, будет конвертирован в использование React.createElement(). Обычно вы не вызываете, React.createElement()если используете JSX.

cloneElement() 

React.cloneElement(
  element,
  [props],
  [...children]
)
  }

Клонировать и вернуть новый элемент React, используя element в качестве отправной точки. Получившийся элемент будет иметь внутренности оригинального элемента с новыми, неглубоко вложенными. Новые элементы-дети заменят существующих. key и ref из исходного элемента будут сохранены.

isValidElement() 

React.isValidElement(object)

Проверяет объект как элемент React. Возвращает true или false.

React.Children

React.Children предоставляет утилиты для работы с this.props.children непрозрачной структурой данных.

React.Children.map 

React.Children.map(children, function[(thisArg)])

Вызывает функцию для каждого дочернего элемента, содержащегося внутри children с this c установленным значением thisArg. Если children это фрагмент или массив, он будет пройден: функция никогда не будет передана объектам контейнера. Если дети null или undefined, возвращает null или undefined реже чем массив.

React.Children.forEach 

React.Children.forEach(children, function[(thisArg)])

Подобна React.Children.map() но не возвращает массив.

React.Children.count 

React.Children.count(children)

Возвращает общее количество компонентов в children, равное количеству раз, когда обратный вызов был передан map или был вызван forEach.

React.Children.only 

React.Children.only(children)

Проверяет, что children имеет только один дочерний элемент (элемент React) и возвращает его. В противном случае этот метод выдает ошибку.

React.Children.toArray 

React.Children.toArray(children)

Возвращает структуру данных как плоский массив с ключами, назначенными каждому ребенку. Полезно, если вы хотите манипулировать коллекциями детей в ваших методах рендеринга, особенно если вы хотите изменить порядок или this.props.children перед его передачей.

React.Fragment

Позволяет возвращать несколько элементов в render() методе, не создавая дополнительный элемент DOM: 

render() {
  return (
    <React.Fragment>
      Some text.
      <h2>A heading</h2>
    </React.Fragment>
  );
}

Вы также можете использовать его с сокращенным <></>синтаксисом.

React.createRef

React.createRef создает ссылку ref, которая может быть присоединена к элементам React через атрибут ref. 

class MyComponent extends React.Component {
  constructor(props) {
    super(props);

    this.inputRef = React.createRef();
  }

  render() {
    return <input type="text" ref={this.inputRef} />;
  }

  componentDidMount() {
    this.inputRef.current.focus();
  }
}

React.forwardRef

React.forwardRef создает компонент React, который перенаправляет атрибут ref, который он получает, на другой компонент ниже в дереве. Этот метод не очень распространен, но особенно полезен в двух сценариях:

  • Пересылка ссылок на компоненты DOM
  • Пересылка ссылок в компонентах более высокого порядка

React.forwardRef принимает функцию рендеринга в качестве аргумента. React вызовет эту функцию с помощью props и ref двумя аргументами. Эта функция должна возвращать узел React. 

const FancyButton = React.forwardRef((props, ref) => (
  <button ref={ref} className="FancyButton">
    {props.children}
  </button>
  ));

  const ref = React.createRef();
  <FancyButton ref={ref}>Click me!</FancyButton>;

В приведенном выше примере React передает ref данный <FancyButton ref={ref}> элемент в качестве второго аргумента функции рендеринга внутри React.forwardRef. Эта функция рендеринга присоединяет React.forwardRef к <button ref={ref}> элементу. В результате после того, как React присоединяет ref, он ref.current укажет непосредственно на экземпляр элемента DOM.
